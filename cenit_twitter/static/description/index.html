Featuring...

Data definitions

Various Schemas that will allow you to map your Odoo models as Twitter's.

  • Tweets
  • Direct messages
  • Followers
  • Lists
  • List members
  • List subscribers
  • Friendships
  • Favourites
  • Blocks.

Set ups

Several Webhooks where you can send or get data.

  • Status Update
  • Status Retweets
  • Status Mentions Timeline
  • Status User Timeline
  • Status Home Timeline
  • Direct Messages New

A handful of Events and Translators to decide when and how perform transformations.

Events:

  • Direct Message on created_at
  • Tweet on created_at
  • Retweet

Translators:

  • Export Tweet
  • Make Tweet ID
  • Import Tweet
  • Export Direct Message
  • Import Direct Message

Out of the Box Flows so you can focus on what's really important.

  • Tweet Send
  • Status Retweet
  • Direct Message Send

First steps

There's really only one thing you need to do: Map your models

  • Go to the Cenit IO Integrations menu.
  • Create a new Data type
    • Name it as you will.
    • Select the model you wish to send over to Twitter (say 'Product Template').
    • Select the 'Twitter' library
    • and the specific schema you want to use (say 'tweet.json').
    • On the Mappings tab set how you want your products to translate to tweets
    • Save your work.

That's it! Everytime you create or modify a Product a tweet will be posted on your Twitter account.

For more on mapping check the Documentation tab of the Cenit IO Integrations Client.
